Transaktionen & ACID

Worum geht's?

Datenbanken müssen zuverlässig arbeiten, auch wenn viele Nutzer gleichzeitig schreiben. Transaktionen fassen mehrere SQL-Schritte zusammen. Die ACID-Regeln definieren, wie stabil diese Abläufe sind.

Das Wichtigste in Kürze

  • Transaktion: Gruppe von Befehlen, die ganz oder gar nicht ausgeführt werden.
  • A - Atomicity: Alles klappt oder es wird komplett zurückgerollt.
  • C - Consistency: Regeln und Constraints bleiben immer gültig.
  • I - Isolation: Gleichzeitige Transaktionen beeinflussen sich nicht.
  • D - Durability: Nach dem Commit bleiben Daten auch nach Ausfall erhalten.
  • Primärschlüssel identifizieren Datensätze eindeutig.
  • Fremdschlüssel verweisen auf Datensätze in anderen Tabellen und sichern Beziehungen.

Beispiel

BEGIN;
UPDATE konto SET saldo = saldo - 50 WHERE id = 1;
UPDATE konto SET saldo = saldo + 50 WHERE id = 2;
COMMIT;

Beide Updates gehören zusammen. Tritt ein Fehler auf, sorgt ein ROLLBACK dafür, dass kein Geld verschwindet.
