Grundprinzipien der OOP

Worum geht's?

Die vier Grundprinzipien helfen dir, Programme sauber zu strukturieren. Wenn du sie anwendest, bleibt dein Code verständlich und leicht erweiterbar. Du erkennst schnell, wo welche Funktion hingehört.

Das Wichtigste in Kürze

  • Abstraktion: Du zeigst nur, was wichtig ist, und versteckst Details.
  • Kapselung: Private Felder schützen Daten; Getter/Setter geben kontrollierten Zugriff.
  • Vererbung: Eine Unterklasse übernimmt Eigenschaften der Oberklasse und kann erweitern.
  • Polymorphie: Gleiche Methode, verschiedene Umsetzung – nützlich für Austauschbarkeit.
  • public, private, protected steuern Sichtbarkeit und Zugriff.
  • override kennzeichnet eine Methode, die Verhalten in der Unterklasse anpasst.
  • Schnittstellen beschreiben nur Methoden, aber keine Felder.

Beispiel

class Tier { public void geraeusch(){ System.out.println("..." ); } }
class Hund extends Tier { @Override public void geraeusch(){ System.out.println("Wuff"); } }

Die Methode geraeusch() wird in der Unterklasse überschrieben und liefert ein eigenes Verhalten.
