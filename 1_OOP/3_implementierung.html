Von UML zum Code

Worum geht's?

Nach dem Modellieren schreibst du passenden Code. UML hilft dir, Attribute und Methoden zu planen. Beim Implementieren achtest du auf Konstruktoren, Datentypen und klare Rückgabewerte.

Das Wichtigste in Kürze

  • Jede Klasse aus dem UML-Diagramm wird als echte Klasse im Code angelegt.
  • Attribute aus dem Diagramm werden zu Feldern mit Typ, z. B. String oder int.
  • Konstruktoren setzen die Startwerte der Objekte.
  • Methoden aus dem Diagramm erhalten Signatur und Rückgabewert.
  • Vererbung wird mit extends (Java) oder : (C#) umgesetzt.
  • Interfaces werden mit implements bzw. interface programmiert.
  • Kleine Tests im main oder mit JUnit prüfen, ob alles funktioniert.

Beispiel

class Person { private String name; public Person(String n){this.name=n;} public String getName(){return name;} }
class Student extends Person { public Student(String n){ super(n); } }

Konstruktoren setzen den Namen beim Erstellen. Student nutzt super(n), um den Elternkonstruktor auszuführen.
