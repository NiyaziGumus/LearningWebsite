UML-Klassendiagramme lesen

Worum geht's?

Klassendiagramme zeigen, wie Klassen zusammenarbeiten. Du erkennst Attribute, Methoden und Beziehungen auf einen Blick. So planst du dein Programm, bevor du Code schreibst.

Das Wichtigste in Kürze

  • Rechtecke stehen für Klassen und enthalten oben den Namen.
  • Pfeile mit leerer Spitze bedeuten Vererbung (Generalisierung).
  • Eine einfache Linie ist eine Assoziation, oft mit Multiplizität 1..*, 0..1 usw.
  • Aggregation wird mit einer leeren Raute dargestellt, Komposition mit einer gefüllten.
  • Attribute haben Sichtbarkeiten wie + (public) oder - (private).
  • Methoden werden mit Parametern und Rückgabetyp wie +zeige() notiert.
  • Notizen oder Interfaces kannst du als gestrichelte Elemente hinzufügen.

Beispiel

classDiagram
  class Kurs {
    -String titel
    +String getTitel()
  }
  class Teilnehmer {
    -String name
    +String getName()
  }
  class OnlineKurs {
    +String url
  }
  Kurs o-- Teilnehmer : nimmt teil
  OnlineKurs --|> Kurs

Die Komposition o-- zeigt, dass ein Kurs mehrere Teilnehmer enthält. OnlineKurs erbt Eigenschaften von Kurs.
