OOP – Überblick

Worum geht's?

Objektorientierte Programmierung beschreibt Programme mit Klassen und Objekten. Du zerlegst Probleme in kleine Bausteine, die klar benannt werden. OOP hilft dir, Code wiederzuverwenden und besser zu strukturieren.

Das Wichtigste in Kürze

  • Klassen beschreiben, welche Eigenschaften und Methoden ein Objekt hat.
  • Objekte sind konkrete Ausprägungen einer Klasse, zum Beispiel ein bestimmter "Student".
  • UML-Diagramme helfen, Klassen und Beziehungen vor dem Programmieren zu planen.
  • Die vier Grundprinzipien heißen Abstraktion, Kapselung, Vererbung und Polymorphie.
  • Entwurfsmuster bieten Lösungsrezepte für typische OOP-Probleme.
  • Sauberer Code nutzt eindeutige Namen und kurze Methoden.

Beispiel

class Kurs { private String titel; public Kurs(String t){titel=t;} public String getTitel(){return titel;} }

Die Klasse Kurs besitzt ein Feld und eine Methode, die den Titel liefert.
