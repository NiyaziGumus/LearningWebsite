Factory & Observer verstehen

Worum geht's?

Entwurfsmuster liefern erprobte Lösungen für typische Probleme. Das Factory-Muster erzeugt Objekte, ohne dass der Aufrufer Details kennen muss. Das Observer-Muster informiert mehrere Empfänger über Änderungen.

Das Wichtigste in Kürze

  • Factory kapselt die Erstellung von Objekten hinter einer Methode.
  • Der Aufrufer fragt nur nach einem Produkt und bekommt die richtige Variante zurück.
  • Observer trennt Sender (Subject) und Empfänger (Observer) locker.
  • Observer melden sich an und werden benachrichtigt, wenn sich der Zustand ändert.
  • UML zeigt beim Factory-Muster meist eine Methode create() im Creator.
  • Beim Observer siehst du eine 1:n-Beziehung vom Subject zu vielen Observern.
  • Muster helfen, Code erweiterbar und testbar zu machen.

Beispiel

Mini-UML (Textform)

Factory --> Produkt
Subject o-- Observer
Observer : +update()

Java-Snippet

class ProduktFactory {
  static Produkt create(String typ){ return switch(typ){ case "A" -> new ProduktA(); default -> new ProduktB(); }; }
}
interface Beobachter { void update(); }
class Anzeige implements Beobachter { public void update(){ System.out.println("Neu!"); } }

Die Factory erzeugt unterschiedliche Produkte anhand eines Typs. Der Observer reagiert auf Updates ohne direkte Kopplung.
