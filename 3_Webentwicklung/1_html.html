HTML-Grundlagen

HTML beschreibt die Struktur einer Webseite. Du ordnest Inhalte in Überschriften, Absätze oder Listen. Formulare erfassen Daten von Nutzenden mit Feldern und Buttons.

  • Jede HTML-Seite startet mit <!DOCTYPE html> und einem <html>-Element.
  • Der <head> enthält Meta-Daten, Titel und CSS-Verlinkungen.
  • Der <body> enthält sichtbare Inhalte.
  • Formulare nutzen <form> mit Eingabefeldern wie <input type="email">.
  • <label> verbindet Text mit Eingaben über das for-Attribut.
  • <select> und <textarea> geben Wahl- und Freitextfelder.
  • Semantische Tags wie <header> oder <nav> verbessern Zugänglichkeit.

<form>
  <label for="mail">E-Mail</label>
  <input id="mail" type="email" required>
  <button type="submit">Senden</button>
</form>

Das Label weist auf das Eingabefeld hin. Das required-Attribut sorgt für eine Pflichtangabe.

  • Formulare mit CSS gestalten
  • Formular-Daten sicher verarbeiten
  • Formular-Daten in Tabellen speichern
  • Serverlogik für Formulare planen

HTML Grundgerüst – kurz & prüfungsfit

1) Minimal (passt fast immer)

<!doctype html>
<html lang="de">
<head>
  <meta charset="utf-8">
  <title>Seitentitel</title>
</head>
<body>
  <!-- Inhalt -->
</body>
</html>

Erklärung:

  • <!doctype html> = HTML5 aktivieren.
  • lang="de" = Seitensprache Deutsch (Screenreader/SEO).
  • meta charset = Zeichensatz UTF‑8 (Umlaute korrekt).
  • title = Titel im Browser‑Tab.

2) Empfohlen (responsive + Dateien einbinden)

<!doctype html>
<html lang="de">
<head>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Seitentitel</title>
  <link rel="stylesheet" href="assets/css/main.css">
  <script src="assets/js/main.js" defer></script>
  <link rel="icon" href="/favicon.ico">
</head>
<body>
  <header>Header</header>
  <main>Hauptinhalt</main>
  <footer>Footer</footer>
</body>
</html>

Erklärung (kurz):

  • viewport = Skaliert auf Handy/Tablet (responsive).
  • link rel="stylesheet" = CSS laden.
  • script defer = JS laden, aber erst nach dem HTML ausführen.
  • favicon = kleines Icon im Tab.
  • header/main/footer = semantische Bereiche (Struktur & Barrierefreiheit).

HTML‑Formulare

1) Was ist ein Formular?

Ein Formular sammelt Nutzereingaben und schickt sie an eine Ziel‑Adresse.

<form action="/absenden" method="post">
  ... Eingabefelder ...
  <input type="submit" value="Absenden">
</form>

2) Wichtige <form>-Attribute

  • action: Wohin werden die Daten gesendet? (z. B. /absenden, verarbeite.php)
  • method: Wie werden Daten gesendet?
    • get → Daten in der URL, z. B. /absenden?name=Ali
    • post → Daten im Request‑Body (nicht in der URL)
  • enctype: Datenformat (für Datei‑Upload multipart/form-data)
<form action="/suche" method="get"> ... </form>
<form action="/upload" method="post" enctype="multipart/form-data"> ... </form>

3) Beschriftungen mit <label>

  • Zweck: Klick auf den Text fokussiert das Feld / setzt Häkchen.
  • Wichtigstes Attribut: for → zeigt auf die id des Eingabefelds.
<label for="email">E‑Mail:</label>
<input type="email" id="email" name="email">

(Alternativ darf das <input> auch im <label> stehen.)

Nützliche globale Attribute am <label>: id, class, title, style.

4) Wichtige Eingabe‑Tags

  • <input> – viele Typen (siehe unten)
  • <textarea> – mehrzeiliger Text
  • <select> + <option> – Dropdown
  • <fieldset> + <legend> – Felder logisch gruppieren

Häufige <input type>-Werte: text, password, number, email, date, radio, checkbox, file, hidden, submit, reset, button

5) Radio‑Buttons (genau eine Auswahl)

Regel: Alle Radios, die zusammengehören, müssen denselben name haben.

<p>Lieferoption:</p>
<label><input type="radio" name="lieferung" value="schnell"> Schnell</label>
<label><input type="radio" name="lieferung" value="guenstig" checked> Günstig</label>
  • Beim Absenden kommt z. B. lieferung=guenstig.
  • checked setzt eine Vorauswahl.

6) Checkboxen (mehrere Auswahlen möglich)

<p>Extras:</p>
<label><input type="checkbox" name="extras" value="versicherung"> Versicherung</label>
<label><input type="checkbox" name="extras" value="express"> Expressversand</label>
<label><input type="checkbox" name="extras" value="geschenk"> Geschenk</label>

Mehrere Haken sind erlaubt. Beim Senden erscheinen mehrere Werte mit demselben Namen (z. B. extras=versicherung&extras=express).

7) Dropdowns (<select>) – eine oder mehrere Auswahlen

Einzelauswahl

<select name="land">
  <option value="de">Deutschland</option>
  <option value="tr">Türkei</option>
  <option value="ch">Schweiz</option>
  <option value="at">Österreich</option>
</select>

Mehrfachauswahl

<select name="laender" multiple size="4">
  <option value="de">Deutschland</option>
  <option value="tr">Türkei</option>
  <option value="ch">Schweiz</option>
  <option value="at">Österreich</option>
</select>

multiple erlaubt mehrere Markierungen (Strg/Cmd‑Klick). size zeigt mehrere Zeilen an.

8) Textarea (mehrzeiliges Feld)

<label for="nachricht">Nachricht:</label>
<textarea id="nachricht" name="nachricht" rows="4" cols="40" placeholder="Ihr Text..."></textarea>

Wichtige Attribute: name, id, rows, cols, placeholder.

9) Gruppen mit <fieldset> & <legend>

<fieldset>
  <legend>Adresse</legend>
  <label for="plz">PLZ:</label>
  <input id="plz" name="plz" type="text">
</fieldset>

Semantisch korrekt und barriereärmer als nur Überschrift‑Text.

10) GET vs. POST – wann was?

  • GET: Lesen/Suchen/Filtern; Lese‑Operationen; URL teilbar.
  • POST: Anlegen/Ändern; Passwörter/Formulare; Datei‑Upload.

Beispiel‑Darstellung

  • GET sendet: /absenden?username=Ali&alter=20
  • POST sendet im Body (nicht sichtbar in der URL)

11) Häufige Fehler in Prüfungen

  • name vergessen → Wert kommt nicht an.
  • Radio‑Buttons mit verschiedenen name-Werten → mehrere gleichzeitig wählbar (falsch).
  • Datei‑Upload ohne enctype="multipart/form-data".
  • <label for="..."> zeigt nicht auf die korrekte id.

12) Mini‑Vorlagen (zum Abschreiben)

Form‑Skelett

<form action="/absenden" method="post">
  ...
  <input type="submit" value="Absenden">
</form>

Radio‑Gruppe

<label><input type="radio" name="x" value="a" checked> A</label>
<label><input type="radio" name="x" value="b"> B</label>

Checkboxen

<label><input type="checkbox" name="opt" value="1"> Opt 1</label>
<label><input type="checkbox" name="opt" value="2"> Opt 2</label>

Select (einfach)

<select name="auswahl">
  <option value="1">Eins</option>
  <option value="2">Zwei</option>
</select>

Textarea

<label for="t">Text:</label>
<textarea id="t" name="t" rows="4" cols="40"></textarea>
