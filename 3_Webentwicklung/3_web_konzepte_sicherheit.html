REST & Sicherheit

Web-Anwendungen tauschen Daten über HTTP aus. REST beschreibt einfache Regeln für Endpunkte. Sicherheit schützt vor Angriffen wie SQL-Injection und sorgt für verschlüsselten Verkehr.

  • REST nutzt eindeutige URLs und HTTP-Methoden wie GET, POST, PUT, DELETE.
  • GET liest Daten, POST legt an, PUT aktualisiert, DELETE entfernt.
  • JSON ist ein häufiges Austauschformat wegen der einfachen Struktur.
  • SQL-Injection entsteht, wenn Eingaben direkt in SQL landen.
  • Prepared Statements binden Parameter und verhindern Code-Einschleusung.
  • HTTPS verwendet TLS, damit Daten unterwegs verschlüsselt bleiben.
  • Cookies und Tokens sollten sicher (HttpOnly, Secure) gesetzt werden.

POST /api/kunden
{
  "name": "Sina",
  "ort": "Leipzig"
}

Der Client sendet neue Kundendaten im JSON-Format. Der Server prüft die Eingaben und nutzt Prepared Statements, bevor sie in die Datenbank kommen.

  • Formularstruktur sicher füllen
  • Sichere Seiten trotzdem schön gestalten
  • Transaktionen mit HTTPS kombinieren
  • Netzwerkprotokolle zum Schutz verstehen
